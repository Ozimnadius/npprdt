Терминал электронного взаимодействия

Аппаратно - программный комплекс с функцией скоростного сканирования документов, доступом к сервисам и личному кабинету ПФР, видеоконсультантом, системой сбора отзывов и мультимедийным справочником.


Многоцелевая платформа, позволяющая решить множество задач клиентских служб ПФР.

В текущую версию Терминала включены такие модули (возможности), как получение извещения о состоянии индивидуального счета, информирования и консультации по вопросам общего характера.

Функциональные возможности ограничены лишь текущими задачами клиентской службы ПФР и развиваются соответственно требованиям и пожеланиям Заказчика.

  • антивандальное сенсорное стекло экран 19’’
  • область сканирования размещена внутри корпуса
  • стеклянная ударопрочная столешница

Назначение терминала:

сокращение затрат на персонал (сотрудников клиентских служб) и содержание помещений без снижения качества обслуживания страхователей
уменьшение очередей
уменьшение времени обслуживания посетителей, путем предоставления сервисов для самостоятельного обслуживания

Основной функционал:

скоростное оптическое сканирование различных типов документов в высоком разрешении

мультимедийный справочно - информационный модуль, включающий справочные материалы ПФР, правовые документы, а также иные фото и видеоматериалы

опрос посетителей (система сбора отзывов), ведение статистики эксплуатации терминала

онлайн видео консультация со специалистом ПФР

интеграция с электронными сервисами ПФР, в том числе:

  • запись на прием и заказ документов
  • направление обращений
  • личный кабинет страхователя
  • пенсионный калькулятор и многое другое

Основные технические характеристики терминала:

Сканирование документов
  • формат до А4
  • скорость получения сканированного документа – не более 0,5 с (для всей области А4)
  • разрешение 300 DPI
Встроенная система отзывов
  • отзыв в текстовом виде (до 800 символов, настраивается)
  • отзыв по 5 заданным критериям (настраивается)
  • аудио отзыв (длительность до 100 секунд)
Автономия
  • устройство не требует дополнительных серверов, ПО, навыков сотрудников
  • весь функционал представлен в едином корпусе
Габаритные размеры
  • 1550(+)(В) х 560(Ш) х 380(Г)
  • вес – 75 кг
Интеграция с популярными сервисами ПФР
  • запись на прием
  • информационные сервисы, включая «видеоконсультант»
  • личный кабинет страхователя
  • электронная очередь
Встроенная система отчетности и сбора статистики
  • возможность удаленного управления и настройки ТСО через WEB интерфейс с любого рабочего места
  • возможность раннего обнаружения возможных проблем
  • статистика использования терминала
  • многоуровневая(администратор / пользователь) система доступа к отчетности
Возможность расширения
  • группа терминалов может быть логически объединена в единое информационное пространство
  • встроенная возможность наблюдения, сбора статистики с группы устройств, используя единый WEB портал
Основные элементы терминала
  • антивандальное сенсорное стекло
  • экран 19’’
  • стеклянная ударопрочная столешница
  • область сканирования размещена внутри корпуса
Технические параметры могут быть заменены производителем на превосходящие без дополнительных уведомлений.
Новое ПО для терминалов самообслуживания предполагает разработку новой архитектуры для решения следующих задач:
  • Кросс-платформенность. Позволит использовать ПО терминала на ОС Windows и Linux.
  • Централизованная настройка и обновление терминалов через сервер.
  • Новая модульная архитектура для сокращения трудозатрат на новые разработки и их интеграцию в другие проекты помимо ПФР

Базовый принцип работы

Локальный сервер

На терминалах будет устанавливаться локальное серверное приложение, работающее с оборудованием терминала, обменивающееся данными с централизованным сервером, выполняющее всю логику, содержащее в себе страницы клиентских меню.

Локальная оболочка (браузер)

Для отображения клиентских страниц (меню клиента), а также раздела настроек, на терминале будет устанавливаться программная оболочка (браузер), работающая под ОС Windows или Linux. Данная оболочка будет загружаться вместо оболочки ОС, предотвращая к ней доступ, а также создавая видимость цельного аппаратно-программного решения.

В этой оболочке в режиме браузера будут отображаться страницы клиента и раздела настроек. Сама по себе оболочка не будет взаимодействовать с оборудование, т.к. эту задачу будет выполнять локальное серверное приложение.

Централизованный сервер.

Останавливается на отдельной машине в районе, отделении, регионе. Позволяет настраивать терминалы через админку с разграничением прав доступа, обновлять ПО, как целиком, так отдельные страницы отображения клиентских окон на терминале.

Централизованный сервер создаст большое поле для новых разработок и больших интеграций с заказчиком, а также позволит предложить заказчику больше услуг по обслуживанию системы.
